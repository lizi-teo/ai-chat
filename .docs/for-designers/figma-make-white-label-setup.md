# White-Label Theming in Figma Make — Designer Setup Guide

This guide walks you through setting up a client-branded prototype in Figma Make using the `@lizi-teo/ai-chat-ui` component library. You will clone the components, extract the client's brand colours, and have an AI agent remap the CSS so every component automatically picks up the client's look.

---

## Before You Start

**What this library is**
A white-label AI chat UI kit built in React + Tailwind CSS. Every colour, radius, shadow, and spacing value is driven by a CSS variable — swapping a client's brand in is entirely a CSS job, no component editing needed.

**Key dependencies designers should know about**
- **Motion** (`motion` / Framer Motion) — components have built-in animation. Do not remove or stub motion imports or animations will break.
- **shadcn/ui** — the Button component and design tokens follow shadcn/ui's semantic naming conventions (e.g. `--primary`, `--muted`, `--destructive`). This matters in Step 3.
- **Tailwind CSS v4** — utility classes are generated from CSS variables, not hardcoded values.

## Step 1 — Clone the Library into Figma Make

Open a new Figma Make project. Paste the following prompt into the Figma Make chat:

---

```
Clone the following GitHub repository into this Figma Make project:
https://github.com/lizi-teo/ai-chat.git

Import ONLY these paths — do not clone the full repo:
- components/         (all component files)
- app/theme.css       (design tokens and per-client theme classes)
- app/globals.css     (Tailwind CSS wiring — do not edit this file)
- lib/tokens.css      (token reference — read-only, comments only)

Important notes for rendering:
1. This library uses the `motion` package (Framer Motion v12) for animations.
   Do not remove or replace motion imports — components will break without them.
2. The Button component and all design tokens follow shadcn/ui semantic naming
   (--primary, --secondary, --muted, --accent, --destructive, --border, --ring, etc.).
3. All colours and spacing are CSS custom properties — never hardcoded.
   Do not add hex values or Tailwind colour utilities directly.

After cloning, render the demo entry point so components are visible on the canvas.
The app shell is in the `demo/` folder if you need a starting render target.
```

---

Wait for Figma Make to finish cloning and rendering. You should see the base component library in its default "clean look" theme (cool off-white palette).

---

## Step 2 — Extract the Client's Brand Colours

You need to collect the client's brand CSS before the AI can remap it. Two ways to do this:

**Option A — Browser DevTools (recommended)**
1. Open the client's live website in Chrome or Safari.
2. Open DevTools → Elements tab.
3. Click the `<html>` or `<body>` element and look at the Computed styles panel for any `--` CSS custom properties. Copy everything you find.
4. Also note the values for: primary brand colour, background colour, text colour, any button colours, any border/divider colours, and the corner radius used on cards and buttons.

**Option B — Copy the stylesheet**
1. In DevTools → Network tab, filter by "CSS".
2. Find the main stylesheet, open it, and search for `:root {` or `--color` to find their token definitions.
3. Copy the relevant section.

Keep these values ready for Step 3.

---

## Step 3 — Map Client Colours to Semantic Token Names

The library expects colours to be named using shadcn/ui's semantic convention, not generic names like `brand-blue`. You need an AI agent to do this translation for you.

Open **GitHub Copilot Chat**, **Claude Code**, or any AI assistant. Paste this prompt, filling in the `[CLIENT COLOURS]` section with what you collected in Step 2:

---

```
I have a white-label React component library that uses shadcn/ui's semantic CSS variable
naming convention. I need to map a client's brand colours to the correct semantic token names.

The library's semantic variables are:
  --background         (page/app background)
  --foreground         (primary text on background)
  --card               (card surface)
  --card-foreground    (text on card)
  --primary            (main brand colour — buttons, links, active states)
  --primary-foreground (text on primary colour)
  --secondary          (secondary actions, subtle highlights)
  --secondary-foreground
  --muted              (disabled states, placeholders, subtle backgrounds)
  --muted-foreground   (secondary/subdued text)
  --accent             (hover states, selected items)
  --accent-foreground
  --destructive        (errors, warnings, delete actions)
  --destructive-foreground
  --border             (dividers, input borders)
  --input              (input field background)
  --ring               (focus ring colour)
  --radius             (border-radius for cards and buttons, e.g. 0.5rem)

The client's brand colours are:
[CLIENT COLOURS — paste what you collected from their website here]

Please output a single CSS class block using this exact format:

.theme-[clientname] {
  --background: ;
  --foreground: ;
  --card: ;
  --card-foreground: ;
  --primary: ;
  --primary-foreground: ;
  --secondary: ;
  --secondary-foreground: ;
  --muted: ;
  --muted-foreground: ;
  --accent: ;
  --accent-foreground: ;
  --destructive: ;
  --destructive-foreground: ;
  --border: ;
  --input: ;
  --ring: ;
  --radius: ;
}

Use oklch() colour format where possible for perceptual consistency.
If a semantic value cannot be inferred from the client colours, derive a sensible
default that respects their palette (e.g. lighten the primary by 80% for --muted).
Replace [clientname] with a lowercase kebab-case version of the client's name.
```

---

Copy the `.theme-[clientname] { … }` block that the AI returns. You will use it in Step 4.

---

## Step 4 — Apply the Client Theme in Figma Make

Go back to Figma Make. Paste this prompt, replacing `[PASTE CSS BLOCK HERE]` with the full `.theme-[clientname] { … }` block from Step 3:

---

```
I have a new client theme CSS block to add to this project.

1. Open app/theme.css and append the following class at the end of the file,
   after all existing .theme-* classes. Do not modify the :root block or any
   existing theme classes — only add the new one:

[PASTE CSS BLOCK HERE]

2. Find the root wrapper element in the demo app (likely in demo/App.tsx or
   demo/index.tsx) and add the class name that matches the new .theme-* class
   you just added (e.g. className="theme-clientname") to the outermost div.

3. Reload the canvas so the components re-render with the new theme.

The components use CSS custom properties throughout — once the class is applied
at the root, every component should automatically pick up the client's colours
without any component-level edits.
```

---

The canvas should now show all components rendered in the client's brand colours.

## Troubleshooting

**Components look unstyled / all white**
The `globals.css` `@theme inline` mappings may not have loaded. Make sure Figma Make imported `app/globals.css` and that it has not been edited or removed.

**Colours are not changing after applying the theme class**
Check that the class name on the root div exactly matches the CSS class name (e.g. `theme-acme` must match `.theme-acme { }`). Class names are case-sensitive.

**Animations are missing or broken**
The `motion` package must be present. Ask Figma Make to check that `motion` is listed in the project's dependencies and that no motion imports were removed during cloning.

**A token is missing from the mapped CSS**
Go back to Step 3 and ask the AI agent to fill in the missing token, specifying what kind of colour it should be (e.g. "derive --ring as a semi-transparent version of --primary").

## Quick Reference — Token Cheat Sheet

| Token | What it controls |
|---|---|
| `--primary` | Buttons, active state, links |
| `--background` | App background |
| `--card` | Message bubbles, cards, panels |
| `--muted` | Placeholder text, disabled states |
| `--border` | Input borders, dividers |
| `--radius` | Corner rounding on all components |
| `--destructive` | Error states, delete confirmations |
| `--ring` | Keyboard focus outline |
